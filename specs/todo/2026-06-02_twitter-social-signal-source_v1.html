1. Hypothesis

Tweets from high-impact accounts (Trump, Elon Musk, cabinet members, Fed officials) and breaking news alerts cause rapid, predictable price movements in specific tickers within seconds to minutes. By monitoring these feeds in real-time, classifying tweet sentiment and market relevance, and mapping them to tradeable tickers, we can generate options signals that capture the initial move before the broader market reacts.

Expected outcome: Identify 2-5 high-conviction event-driven trades per week with 15-30% average gain on 0DTE options, reacting within 30-90 seconds of a market-moving tweet. These signals would be additive to the existing ML pattern scanner (different alpha source).

2. Background & Motivation

The current OptionsOwl pipeline sources signals from Discord (Neverland Pirates) and an ML pattern scanner. Both are reactive to price action — they detect moves already in progress. Twitter/social media signals represent a fundamentally different alpha source: event-driven catalysts that cause moves rather than react to them.

  • Political tweets: Trump tariff announcements, trade deal mentions, company callouts (historically moved SPY 0.5-2% in minutes)
  • Elon Musk: TSLA-specific tweets, crypto mentions, regulatory comments
  • Fed officials: Rate hints, economic outlook (SPY, QQQ, IWM)
  • Breaking news: Earnings leaks, FDA approvals, antitrust actions, M&A rumors
  • Cabinet/policy: Commerce Sec (tariffs), Treasury Sec (debt ceiling), AG (antitrust)

The edge is speed + classification: most retail traders see these tweets minutes later via news aggregators. A direct Twitter stream with ML classification can react in under 60 seconds.

4.3 Key Technical Decisions

  • Twitter API tier: X/Twitter API v2 Pro ($5000/mo) for filtered stream, or Basic ($100/mo) for polling. Alternative: use free news aggregator APIs (Benzinga, NewsAPI) as proxy
  • Classification: Claude API for tweet classification (already have Anthropic key) vs rules-based keyword matching (faster, cheaper, good enough for high-impact accounts)
  • Latency target: Tweet → signal in <30 seconds. WebSocket stream preferred over polling
  • False positive handling: High-impact accounts post many irrelevant tweets. Classification must filter aggressively — only trade tweets that mention markets, companies, policy, or economics

7. Risks & Considerations

  • API cost: Twitter Pro tier is $5000/mo — need to validate alpha before committing. Start with free alternatives (snscrape, news APIs)
  • Latency: If everyone is trading the same tweets, the edge disappears. Need sub-30s reaction time
  • False positives: High-impact accounts post constantly. Most tweets are irrelevant. Classification accuracy is critical
  • Account suspension: Twitter API terms change frequently. Need fallback data sources
  • Regulatory risk: Trading on government officials' public statements is legal, but front-running non-public info is not. Only use publicly available tweets
  • Crowded trade: Many hedge funds already do this. The alpha may be fully captured by the time retail can react
