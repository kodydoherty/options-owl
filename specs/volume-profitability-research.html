2. Testing Methodology

Scientific Method: Change ONE variable at a time from the baseline. Run the SAME 60-day backtest window. Compare P&L, WR, trades/day, Sharpe, and max drawdown. Only variables that improve results are candidates for combination testing.

Test Execution Protocol

  1. Modify the single parameter in the backtest script (create a copy or use CLI flags)
  2. Run: python scripts/backtest_gold_standard.py --days 60 (or backtest_combined.py)
  3. Record: Total P&L, Win Rate, Trade Count, Trades/Day, PF, Sharpe, MaxDD
  4. Compare to baseline: Did it improve trades/day WITHOUT destroying WR/PF?
  5. Pass criteria:
    • P&L ≥ baseline (or close, if volume increase is significant)
    • WR ≥ 75% (we accept WR degradation if PF stays ≥ 3.0)
    • Trades/day improvement is meaningful (≥ 0.5 more trades/day)
  6. Log result in the results table below

3. Entry Pipeline Tests (14 Isolated Variables)

Each test modifies one parameter in the entry pipeline while keeping all exit and portfolio settings identical to baseline.

E1: Extend Scan Window to 2:00 PM

Variable: SCAN_END_MIN = 270 (was 90) Hypothesis: More scanning time = more entries VOLUME TEST

What changes: Pattern model scans 9:35 AM – 2:00 PM instead of stopping at 11:00 AM. Same ML thresholds, same everything else.

Risk: Midday chop (11:00–1:00) historically has lower win rates. Pattern model was trained on morning data — may not generalize to afternoon.

Why test: The current 90-minute window is the #1 volume bottleneck. If afternoon signals have even moderate quality, this could double trade count.

SCAN_END_MIN = 270  # was 90
# Everything else identical to baseline

E2: Extend Scan Window to 3:45 PM (Full Day)

Variable: SCAN_END_MIN = 375 (was 90) Hypothesis: Power hour (2:30–3:45) has high volatility VOLUME TEST

What changes: Full-day scanning. Tests whether power hour produces quality signals with our existing ML models.

Risk: Gamma decay intensifies after 2PM for 0DTE. ML model trained on morning data.

Why test: Power hour volatility is real — if the pattern model fires good signals there, we get a free second trading window.

SCAN_END_MIN = 375  # was 90

Variable: PATTERN_THRESHOLD = 0.80 (was 0.85) Hypothesis: Looser filter lets in more good trades VOLUME TEST

Risk: 0.80–0.85 band may contain false positives that the entry timing model can't fully filter.
