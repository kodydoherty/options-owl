1. Goal

Train a LightGBM model specifically on PUT option chain data to predict "will this PUT gain 20%+ in 60 minutes?" The existing pattern model was trained on CALL data only. While Approach A fixes the data pipeline so the CALL model at least sees PUT chain data, a dedicated PUT model should outperform it because PUT market dynamics differ from CALLs (faster moves, higher IV, wider spreads, different volume patterns).

3. Training Design

3.1 Label Definition

# Label: did this PUT option gain >= 20% within 60 minutes?
# Shorter window than CALLs (120min) because PUTs are scalps
label = 1 if max(premium[t:t+60]) / premium[t] >= 1.20 else 0

3.2 Feature Set (30 features)

CategoryFeaturesDescription
Premium dynamics premium_mom_5m, premium_mom_10m, premium_mom_15m, premium_range_pct PUT premium momentum at multiple lookbacks
Bid/ask spread_pct, bid_size_ratio, bid_trend_5m Liquidity and demand pressure
Greeks delta, iv, theta, vega, iv_mom_5m, delta_mom_5m Option sensitivity (using PUT-native values, not abs())
Underlying underlying_mom_5m, underlying_mom_10m, underlying_mom_15m, underlying_mom_30m Stock decline rate at multiple timeframes
Volume volume_avg_30m, volume_current_vs_avg, volume_trend Activity and demand surge
Market context vwap_deviation_pct, price_position_in_range, day_open_to_now_pct Where we are in the day's range
PUT-specific put_iv_vs_call_iv, put_volume_vs_call_volume, underlying_decline_rate_5m, spy_correlation_5m Fear indicators, correlation with broad market
Time/ticker minutes_since_open, day_of_week, ticker_encoded Temporal and categorical features

3.3 Train/Test Split

  • Train: Dec 8 2025 – Apr 30 2026 (~100 days)
  • Validation: May 1 – May 15 2026 (~10 days)
  • Test: May 16 – Jun 5 2026 (~14 days)
Time-based split only. Never random-split financial time series — forward-only prevents look-ahead bias.

5. Success Criteria

  • PUT model AUC ≥ 0.80 on held-out test set — PASS: AUC = 0.8038
  • PUT-only backtest: win rate ≥ 55%, profit factor ≥ 1.5 — PASS: WR 75.6%, PF 1.57
  • PUT model outperforms CALL model on PUT data by ≥ 5% in P&L — PASS: dedicated model $+63K with trail config
  • No degradation of CALL performance (models are separate) — PASS: CALLs $+279K unchanged

6. Risks

  • Overfitting to recent bearish periods: Mitigate with time-based split + early stopping
  • Class imbalance: Most PUTs don't gain 20%. Use scale_pos_weight or focal loss
  • PUT dynamics change with volatility regime: Monitor model degradation monthly, retrain quarterly
