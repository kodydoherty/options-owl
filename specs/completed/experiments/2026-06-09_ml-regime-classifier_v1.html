1. Hypothesis

A LightGBM classifier trained on SPY intraday features can predict market regime (BULLISH/BEARISH/CHOPPY) with >80% directional accuracy within the first 30 minutes of trading, enabling earlier and more accurate CALL/PUT slot allocation than the current rule-based EMA/RSI/ADX detector.

Expected outcome: Directional accuracy at 10:00 AM improves from 57% (rule-based) to >75%, and at 10:30 AM from 74% to >85%. This translates to blocking $1,500+ in counter-trend losses per crash day while preserving winning trend-aligned trades.

2. Background & Motivation

On 2026-06-09 (SPY -2.7%), the bot fired 5 CALL trades into a bear market, losing $1,595 on AMZN and GOOGL CALLs that had no chance. The regime detector exists but was disabled because:

  • The EMA/RSI/ADX classifier needs 22 five-minute bars (~110min) for warmup, making it useless for the critical 9:35-10:30 AM CALL window
  • Gap detection (opening gap vs prior close) tested at only 48% accuracy — worse than a coin flip because gaps frequently reverse intraday
  • First-15-min direction predicts at 69% — decent but noisy

Current rule-based accuracy over 126 days (Dec 2025 – Jun 2026):

The gap between 10:00 (63%) and 11:00 (85%) is the problem zone. An ML model that extracts more signal from early price action could close this gap.

3.2 Test (Experiment)

LightGBM classifier trained to predict daily regime from early-session features. Three output classes: BULLISH (SPY EOD return > +0.3%), BEARISH (EOD < -0.3%), CHOPPY (in between). Predictions at multiple checkpoints (15min, 30min, 60min, 90min).

4.2 Production Integration

# In regime_detector.py
class MLRegimeDetector(RegimeDetector):
    """ML-based regime classifier, replaces rule-based _classify()."""

    def __init__(self, model_path: str, **kwargs):
        super().__init__(**kwargs)
        self._model = lgb.Booster(model_file=model_path)
        self._feature_buffer = []  # rolling feature accumulator

    def _classify(self, indicators: dict) -> RegimeState:
        # Build feature vector from indicators + accumulated history
        features = self._extract_features(indicators)
        probs = self._model.predict([features])[0]
        # probs = [p_bullish, p_bearish, p_choppy]
        max_idx = probs.argmax()
        confidence = probs[max_idx]

        # Only commit if confidence > threshold (otherwise CHOPPY)
        if confidence < 0.6:
            return RegimeState.CHOPPY

        return [RegimeState.BULLISH, RegimeState.BEARISH, RegimeState.CHOPPY][max_idx]

6.3 Why ML Failed

  • Markets are efficient: Early price action (gap, opening drive, prior returns) doesn't predict daily direction with >50% accuracy because these signals are already priced in.
  • Rule-based is reactive, not predictive: The EMA/RSI/VWAP detector reads what IS happening (trend already established), while the ML model tries to PREDICT what WILL happen.
  • Overfitting: Even with heavy regularization (max_depth=4, min_child_samples=50, L1/L2=1.0), the model memorizes training patterns that don't generalize.
  • Gap detection is noise: Gap-down days reversed intraday 52% of the time (March 2026: three -1% gaps all ended +1% up). Gaps are mean-reverting, not directional.

6.4 Data Used

  • 560 trading days of SPY 1-min candles (Apr 2024 – Jun 2026)
  • Sources: Polygon (historical_0dte.db, 509 days) + ThetaData (126 days)
  • Backfilled 422,911 real 1-min bars from Polygon into unified thetadata_options.db

7. Risks & Mitigations

  • Overfitting on 126 days: Mitigate with walk-forward validation, simpler model (fewer trees), regularization, and downloading more historical data
  • Regime shift: Model trained on Dec 2025 – Jun 2026 may not generalize to different market conditions. Retrain quarterly with fresh data.
  • Feature leakage: Ensure no features peek at EOD data. All features must use data available at the checkpoint time.
  • Latency: Model must predict in <100ms. LightGBM inference is sub-millisecond — not a concern.
  • False BEARISH on gap reversals: March 2026 had multiple gap-down days that reversed +1%. The model must learn that gaps alone are not predictive (48% accuracy) and should weight intraday price action more heavily.
