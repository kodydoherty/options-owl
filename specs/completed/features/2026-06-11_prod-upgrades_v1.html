1. Why this exists

The 2026-06-10/11 review + sweeps surfaced a set of validated and in-progress production improvements. This spec tracks what is live, what is OOS-validated and ready to stage, and what still needs multi-window validation before touching real money. 3. Sizing design targets (item 5 — the priority)

  • Make size track conviction. A high-confidence signal should deploy a meaningful fraction of the book, not $442 of $19K. Test a monotonic confidence→budget% curve (replacing the non-monotonic 100/60/95 tiers).
  • Multi-day cap. The hard 2-contract cap on DTE≥1 is the dominant throttle on non-0DTE days. Test scaling it (e.g. by DTE / by premium / disable) — measuring drawdown, since multi-day options can take outsized losses (the reason it exists).
  • Capital utilization. Per-slot = deployable ÷ MAX_CONCURRENT(8). With typically 2–3 concurrent, the book is under-deployed. Test fewer/bigger slots or dynamic sizing off available capital.
  • Risk metric, not just P&L. Bigger size = bigger drawdown. Judge on P&L AND Sharpe / max-DD across multiple windows.

4. Validation protocol (mandatory before any item goes live)

  1. Sweep on the in-sample window (2026-02-01 → 2026-05-20).
  2. Multi-window walk-forward: month-by-month across 2025-09 → 2026-06; a candidate must hold (P&L + max-DD) in most regimes, not just one.
  3. OOS window (2025-09-08 → 2025-12-07, OOS for the primary CALL/PUT models).
  4. Reject in-sample-only winners (this discipline already caught put_bearish and the regime-0.19 threshold traps).
  5. Deploy via rebuild.sh / staggered up -d; verify live container env; keep .env.bak for rollback.

5. Open follow-ups / risks

  • Webull entry no-fills (QQQ #253 today: not filled in 30s → cancelled). Review entry limit aggressiveness / chase logic — missed entries, not losses, but lost edge.
  • confirmed_stop on multi-day trades (SMCI/MSTR ~−32% today): verify multi-day stops use the wider multi-day thresholds, not 0DTE-tight ones.
  • P&L source: use scripts/trade-pnl.py for authoritative P&L (the pnl_dollars column is approximate).
  • Backtest absolute $ is optimistic (window partly in-sample for V3/PUT models); weight relative comparisons + OOS.
