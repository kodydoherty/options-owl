1. Overview

Replace the static premium_cap ($6) and per-ticker OTM distance gates with a single delta-based entry filter. Delta is a market-derived measure that automatically adapts to each ticker's price, IV, and time-to-expiry — eliminating the overfitting risk of hand-tuned dollar thresholds.

2. Problem Statement

The old price gates (premium_cap and otm_distance) were blocking $184,976 in profitable trades over 126 days. Backtest analysis showed:

  • premium_cap ($6): Backtested to find the "optimal" dollar amount — classic curve-fitting. A $6 META option is cheap; a $6 SPY option is expensive. No universal threshold works.
  • otm_distance (per-ticker strikes): The 1/2/3 strike multipliers were hand-tuned. The ML pattern model already captures moneyness — this gate double-filters.
  • Both gates reject trades the ML model already approved, second-guessing it with crude heuristics.

Current state: premium_cap and otm_distance are DISABLED in production (deployed 2026-06-10). Spread gate and min_premium are still active (principled market-microstructure filters).

4. Business Logic

4.1 Delta Gate Rules

  • CALL entries: Reject if delta < 0.15 (far OTM lottery ticket) or delta > 0.70 (deep ITM, overpaying for intrinsic value)
  • PUT entries: Same thresholds on |delta| (PUT delta is negative, so use abs value)
  • Missing delta: SKIP gate (don't block — delta may not be available for all sources)
  • Zero delta: SKIP gate (data quality issue, don't block on bad data)

4.2 Why Delta Works

  • Delta is the probability-weighted measure of an option finishing ITM
  • Delta < 0.15 = <15% chance of expiring ITM — high theta decay, needs large underlying move
  • Delta > 0.70 = deep ITM — premium is mostly intrinsic value, limited leverage
  • Unlike a $6 cap, delta automatically adapts to ticker price, IV environment, and DTE
  • A $6 NVDA option may have delta 0.45 (great entry), while a $6 SPY option may have delta 0.80 (overpaying)

4.3 Data Source for Delta

  • Production (entry pipeline): Read delta from Redis option snapshot (published by harvester with full greeks)
  • Fallback: Polygon REST option quote includes greeks
  • Backtest: Delta available in ThetaData option_greeks table (already loaded by gold standard backtest)
Dependency: The delta gate requires real-time delta data from the harvester via Redis. The harvester already publishes full option snapshots including delta to Redis every 60s. For sub-second delta, the real-time data streaming spec (separate) must be completed first. However, the 60s harvester cycle is sufficient for entry gating — we only need delta at the moment of entry, not continuously.

6. Code Changes

6.1 DeltaEntryGate (pipeline.py)

class DeltaEntryGate(EntryGate):
    """Gate: Reject entries where option delta is outside the sweet spot.

    Delta < min (default 0.15) = far OTM lottery ticket, high theta decay.
    Delta > max (default 0.70) = deep ITM, overpaying for intrinsic value.

    Replaces static premium_cap and otm_distance gates with a single
    market-derived measure that adapts to ticker price, IV, and DTE.

    Backtested: delta 0.15-0.70 = $+491K, $742/trade, 75% WR over 126 days.
    """
    name = "delta_entry"

    async def evaluate(self, ctx: dict[str, Any]) -> GateOutcome:
        settings = ctx.get("settings")
        if not getattr(settings, "ENABLE_DELTA_GATE", False):
            return GateOutcome(self.name, GateResult.SKIP, "delta gate disabled")

        delta = ctx.get("entry_delta")
        if delta is None or delta == 0:
            return GateOutcome(self.name, GateResult.SKIP,
                               "No delta data — skipping")

        abs_delta = abs(delta)
        min_delta = getattr(settings, "DELTA_ENTRY_MIN", 0.15)
        max_delta = getattr(settings, "DELTA_ENTRY_MAX", 0.70)

        if abs_delta < min_delta:
            return GateOutcome(self.name, GateResult.FAIL,
                f"Delta {abs_delta:.3f} < {min_delta} — far OTM lottery ticket")

        if abs_delta > max_delta:
            return GateOutcome(self.name, GateResult.FAIL,
                f"Delta {abs_delta:.3f} > {max_delta} — deep ITM, limited leverage")

        return GateOutcome(self.name, GateResult.PASS,
            f"Delta {abs_delta:.3f} in range [{min_delta}, {max_delta}]")

        contract_key = f"{ticker}:{direction}:{strike}:{expiry}"
        snapshot = await redis_client.get_option_snapshot(contract_key)
        if snapshot and snapshot.get("delta"):
            entry_delta = snapshot["delta"]

  • Production verification: grep logs for "delta_entry" gate outcomes after deploy

8. Security Considerations

  • Delta read from Redis is untrusted external data — validate numeric range (0-1)
  • Gate SKIPs (not blocks) on missing/zero delta — fail-open design prevents data issues from blocking all trades
  • No user input involved — all values from Redis/Polygon API

9. Rollback Plan

Set ENABLE_DELTA_GATE=false in docker-compose.yml and redeploy. Gate returns SKIP when disabled. The old premium_cap and otm_distance gates still exist in code (just disabled) and can be re-enabled independently.
